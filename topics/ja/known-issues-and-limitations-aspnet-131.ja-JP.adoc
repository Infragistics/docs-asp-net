=== UpdatePanel の WebHierarchicalDataGrid と WebHierarchicalDataSource でビジュアル スタジオ デザイナーがフリーズする問題

Microsoft Visual Studio® のデザイナーで UpdatePanel 内で  _WebHierarchicalDataGrid_™ および  _WebHierarchicalDataSource_   を使用する際に、デザイナーが持続的に再描画しフリーズしたようになります。

回避方法

_WebHierarchicalDataSource_   を UpdatePanel コントロールの外側に配置します。

[[_Ref351489288]]

=== WebTab が 要素に追加されるときにタブ幅が正しくない可能性がある問題

_WebTab_   で幅が定義されていないかパーセンテージで定義されている場合、`<TD>` 要素に配置される際に幅が定義されない問題。 _WebTab_   コントロールがタブ幅を正しく計算できないためタブ幅が過度に大きくなる問題。

回避方法

_WebTab_   コントロールでピクセル幅を明示的に構成します。

=== WebDataGrid/ WebHierarchicalDataGrid で IME モードで編集モードがキー押下でトリガーされるよう構成されているの場合、編集モードで入力テキストの最初の文字を誤って解釈する可能性がある問題

WebDataGrid コントロールがアクティブな IME モードの編集モードがキー押下でトリガーされるよう設定されている場合に、ユーザーが最初の文字を入力するとローマ字が表示されることがある問題。

この問題は、IME を解釈するときにブラウザーが実際の `INPUT` 要素を必要とするために発生します。 _WebDataGrid_  / _WebHierarchicalDataGrid_   が`INPUT` とは反対に `<TD>` 要素でデータを表示するため、ブラウザーは `INPUT` が編集に表示 (IME がコンポーネント文字を許可またはコマンドを処理する時) されるまでローマ字を使用します。これによって、最初の入力文字がローマ字でその後に正しいコンポーネント文字になります。

回避方法

`enteredEditMode` イベントから初期の文字をクリアにします。

*JavaScript の場合:*

[source,js]
----
function entered(grid, args) {
          var editor = args.getEditor();
     setTimeout(function () { editor.set_value("") }, 0);
}
----

=== WebDataGrid と WebTab が UpdatePanel 内にある場合にポストバックがメモリ リークを発生することがある問題

UpdatePanel 内の  _WebDataGrid_   と  _WebTab_   コントロールで、非同期ポストバックがメモリリークを発生する可能性があります。

この問題はブラウザーまたは ASP.NET AJAX ライブラリで発生し、Microsoft の GridView および UpdatePanel 内の HTML テーブルでも再現できます。

回避方法

*注:*Infragistics AJAX コントロールを UpdatePanel 以外に配置します。コントロールは定義済みの AJAX 機能があります。

=== WebSchedule の複数日にわたる定期的な予定が週末の場合正しく描画されない問題

_WebSchedule_   コントロールで、定期的な予定が週末を含む複数日にまたがる際に正しく描画されません。

回避方法

*注:*_WebSchedule_   `DetailEventCalenderWebScheduleInfo` オブジェクトの `EnableMultiDayEventBanner` プロパティを _false_ に設定します。

*C# の場合:*

[source,csharp]
----
WebSchedule.DetailEventCalenderWebScheduleInfo.EnableMultiDayEventBanner = false;
----
