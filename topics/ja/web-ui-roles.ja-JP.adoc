= UI ロール

アプリケーション スタイリングを使用してアプリケーションをスタイルするためには、ロールの概念とコントロールとの関連を理解する必要があります。ロールは、特定のカスケード スタイル シート クラスが適用されることになる UI 要素または動作のタイプを示すために使用されます。コントロール間で共有される一般的なロールは複数あります（Hover、Selected、Control など）。しかし、コントロール固有のロールもあります。たとえば、WebTree は Island、Node、Parent、および Leaf などのロールを持っています。

すべてのコントロールのすべてのロールだけでなくすべての共有ロールのリストは長すぎます。さまざまなロール、そして各ロールがコントロールに対してどのような影響があるのかを理解する最も簡単な方法は、Infragistics AppStylist™ for ASP.NET 製品でロールを使用することです。このアプリケーションはすべてのコントロールを、コントロールの使用可能なロールであるサブノードを含む各コントロールのノードとともに表示するためにツリーを使用します。このプログラムの詳細は、「アプリケーションのスタイリング」を参照してください。
