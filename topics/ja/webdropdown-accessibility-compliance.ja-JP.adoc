= WebDropDown アクセシビリティの遵守

弊社の ASP.NET コントロールおよびコンポーネントはすべて、1973 年のリハビリテーション法第 508 条第 1194 部 22 条を遵守しています。以下は、コントロールに関係する第 1194 部 22 条の特定の規則と各規則を遵守する方法を含んだ表です。場合によって、プロパティを設定する必要があります。それ以外の場合には、コントロールがすべての作業を実行するため、ユーザーは何もする必要がありません。

[options="header", cols="a,a"]
|====
|規則|規則を遵守する方法

|(a)
|主要な機能を持つすべての IMG タグは "alt" 属性プロパティが設定されるか、alt 属性値の設定が可能です。

|(c)
|クライアント側イベントを使用すると、現在のノードの状態（アクティブ化されている、選択されている、無効というように）を反映するいかなるコントロールの DOM 要素の属性をユーザーは変更できます。このように色（またはスタイル）によって伝達される情報はスクリーン リーダーで使用できます。

|(d)
|規則 A と C で説明されている属性のために、コントロールのマークアップは関連付けられたスタイル シートを必要とすることなく読み取り可能です。

|(l)
|コントロールのスクリプトがページ マークアップを変更している時、規則 A と C で説明されている属性のために新しく作成されたコンテンツはスクリーン リーダーに準拠します。

|(n)
|フォームに配置されると、展開、縮小、選択、編集、ノードのナビゲーションを含み、マウスおよびキーボードの両方を使用して、コントロールのすべてのパーツはアクセス可能です。

|(p)
|カスタマイズ可能な「データ ロード中」メッセージがサポートされ、開発者はサーバーからデータ フェッチについてユーザーに通知することが可能となります。

|====
