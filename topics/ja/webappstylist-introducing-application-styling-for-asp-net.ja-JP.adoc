= Application Styling for ASP.NET の紹介

ここ数年間、Web ページのデザインは、カスケード スタイル シートの汎用性と能力に依存してきました。これらのスタイル シートは、Web ページのグラフィカルなルールセットを定義するだけでなく、カスケードまたは継承効果にしたがってさまざまなルック アンド フィールを構成および分類します。カスケード スタイルシートが優れているのは、インライン スタイルを使用する必要がなく、div、paragraph、id、または span タグにクラスを割り当てることですべてのスタイルを定義できるところです。アプリケーション スタイリングがスタイル シートの利点を十分に活用できるのはこのためです。

各 {ProductName} コントロール/コンポーネントには、これらのコントロール/コンポーネントがなることができるいくつかの状態があります。たとえば、ボタンは通常の状態、ホバーした状態、または押された状態になることができます。これらの各状態はスタイルシートのクラスで表され、各コントロールはスタイル シート全体で表されます。
